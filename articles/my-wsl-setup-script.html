Motivation

As of late, I find my self purging my WSL distribution and installing it all from fresh. While doing so I backup my .bashrc file and then import it back to source it. After that, I always try and open up VSCode, doing so prompts me to install wget, so I start updating everything just to install wget.

Finally, I can start working on my projects again, let me just run git clone repo and, oh right.. I need to install git, and I need to setup the credential manager to use the Windows one to make my life easier. So I google how to do that and then I can start working on my projects finally!

Well, the more I work with Unix and WSL in particular, the more I see myself having more and more things that I will need to install on a fresh distribution and more and more things that I know I will need.

I have created this page, to have a quick command I (and you) can use to setup WSL after a fresh installation.

People have done it before, hosting the file on their GitHub page. I just make it a bit more available and easier to use for myself.

IMPORTANT NOTE: this script is nothing fancy, and you should always understand a script fully before running it. It will make you a more secure user, and it will elevate your expertise in the long run.

Running the Script

To get the script to run you can download it and run it using the following command. The script itself can be viewed here.

The funny thing is that to use the script you might need to have curl or wget already installed, which is part of what the script does. If you have any one installed just run the following command: 

bash <(curl -sSL https://nubpaws.github.io/files/my-wsl-setup-script/setup_shell.sh)
bash <(wget -qO- https://nubpaws.github.io/files/my-wsl-setup-script/setup_shell.sh)

If you have neither, I would advise install curl and then running the script, so the following command: 

sudo apt-get install curl && bash <(curl -sSL https://nubpaws.github.io/files/my-wsl-setup-script/setup_shell.sh)

What does it do?

The script does the following actions:

  • Runs update, upgrade and autoremove on the system to get everything updated.
  • Installs a bunch of software I find useful:
    • git
    • curl
    • wget
    • build-essential
    • gdb
    • python3
    • python3-pip
    • python3-venv
    • unzip
  • Setups git credential manager assuming the newest version of Git.
  • Adds a bunch of aliases like update-all that updates the system and explorer which opens windows file explorer in the current working directory.
  • Setsup the variable $DESKTOP for the user's desktop folder which is always a nice place to jump to other projects on your Windows machine from.
  • Setups the prompt to look like my prompt. You might not like it so it will be the last line in the ~/.bashrc file to delete.
